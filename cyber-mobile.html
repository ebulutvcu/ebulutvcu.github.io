Objective: To train students towards building a workforce in the area of secure app development. We will specifically consider iOS app development and its security integrations during development. The students will also be provided an opportunity to do internship.

Target Student Population and Pre-requisites:

Our target population is a cohort of students from both VCU and VUU. The program will be open to students with all backgrounds with some prerequisites satisfied. These prerequisites include basic understanding in programming. Students can have a background not only in Computer Science, but also in other related fields e.g., Computer and Electrical Engineering, Information Technology. Our goal is to expose the students who would normally not have much chance to take such a class but have abilities and skills. This will increase the cybersecurity workforce in mobile (iOS) app development industry by leveraging more students to this area.

Timeline and Milestones:

The proposed program will be executed with a 10-week session in Fall 2023 follow up with internships in Spring 2024. Each session will be in person and will consist of 1 hour instruction and another 30-60 min self exercise and assignment completion. The program is open to VCU and VUU students only. Please submit your application here. Tentative Course Content:

  • Major Topics Covered: We will start with the introduction of basic iOS programming fundamentals and frameworks. These include Swift basics using playground: variables, operations, loops, if statements, functions (ii) Object oriented programming, polymorphism, inheritance, (iii) Swift UIKit Framework: Tableview, and Collectionviews, (iv) ViewControllers and Segues, (v) Touches and Gestures, (vi) Core Location, CoreMotion, (vii) Web services, networking, data persistence.
  • Security Aspect: Once the essential iOS programming topics are covered, we will also introduce the topics for securing the data that the app manages, and controlling the access to the apps, using the iOS Security Framework. This framework provides services that can (i) establish a user’s identity (authentication) and then selectively grant access to resources (authorization), (ii) secure data, both on disk and in motion across a network connection, and (iii) ensure the validity of code to be executed for a particular purpose. It is also possible to use lower-level cryptographic resources to create new secure services, however, as this topic can be complicated for targeted student population, we will not include it during this training.
  • Hands-on Assignments: Through the training, there will be assignments which will provide students be able to practice their knowledge. The assignments will generally focus on development of components of a specific app which may have security vulnerabilities. For example, we will look at (i) showing and hiding password fields, (ii) Accessing the private resources securely, (iii) Secure coding of Touch ID authentication and other Touch gestures, (iv) Avoiding Man-in-the-Middle (MiTM) attacks in HTTP (not secure) requests, (v) Preventing screen recording and capturing by attackers, (vi) Protection of keychain, (vii) Protecting data in files included in apps, and (viii) Detection of jail break devices to prevent compromising the app logic.
